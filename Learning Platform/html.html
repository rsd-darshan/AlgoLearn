Course Overview

This course covers various topics in HTML and CSS, including basics, layout techniques, responsive design, and more. Each section provides detailed explanations and examples to help you understand and apply HTML and CSS concepts.

What is HTML?

HTML, or HyperText Markup Language, is the standard markup language used to create web pages. A markup language is a computer language that uses tags to define elements within a document.

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document Title</title>
</head>
<body>
    <h1>This is a Heading</h1>
    <p>This is a paragraph.</p>
</body>
</html>

HTML Elements, Tags, and Attributes

HTML elements are the building blocks of HTML pages. They are represented by tags. HTML tags label pieces of content such as "heading", "paragraph", "table", and so on.

  • <h1> to <h6> - Headings
  • <p> - Paragraph
  • <a> - Anchor (link)
  • <img> - Image
  • <div> - Division
  • <span> - Span
  • <ul> - Unordered List
  • <ol> - Ordered List
  • <li> - List Item
  • <table> - Table

What is CSS?

CSS, or Cascading Style Sheets, is a style sheet language used for describing the presentation of a document written in HTML. CSS describes how HTML elements are to be displayed on screen, paper, or in other media.

body {
    font-family: Arial, sans-serif;
    background-color: #f9f9f9;
}

h1 {
    color: #333;
}

p {
    line-height: 1.6;
}

Basic CSS Selectors

CSS selectors are used to "find" (or select) the HTML elements you want to style. We can divide CSS selectors into five categories:

  • Simple selectors (select elements based on name, id, class)
  • Combinator selectors (select elements based on a specific relationship between them)
  • Pseudo-class selectors (select elements based on a certain state)
  • Pseudo-elements selectors (select and style a part of an element)
  • Attribute selectors (select elements based on an attribute or attribute value)

Forms

HTML forms are used to collect user input. The form element is used to create an HTML form for user input. Form elements include input fields, radio buttons, checkboxes, and more.

<form action="/submit-form" method="post">

Tables

HTML tables allow web developers to arrange data into rows and columns. The table element is used to create a table. A table is divided into rows with the tr tag, and each row is divided into data cells with the td tag.

CSS Positioning

CSS positioning properties allow you to position an element relative to its parent element, its containing element, or the viewport. The position property can take several values:

  • static: default value. Elements are positioned according to the normal flow of the document.
  • relative: elements are positioned relative to their normal position.
  • absolute: elements are positioned relative to their nearest positioned ancestor.
  • fixed: elements are positioned relative to the browser window.
  • sticky: elements are positioned based on the user's scroll position.
.relative {
    position: relative;
    top: 10px;
    left: 10px;
}

.absolute {
    position: absolute;
    top: 10px;
    left: 10px;
}

.fixed {
    position: fixed;
    top: 0;
    width: 100%;
}

.sticky {
    position: sticky;
    top: 0;
    background-color: yellow;
}

CSS Flexbox

CSS flexbox layout allows you to design a flexible responsive layout structure without using float or positioning. The flexbox layout module is designed to lay out elements in one dimension either as a row or a column.

.container {
    display: flex;
    flex-direction: row;
}

.item {
    flex: 1;
    padding: 10px;
}

Flexbox Properties

  • display: flex; - Defines a flex container
  • flex-direction - Defines the direction of the flex container's main axis (row or column)
  • justify-content - Aligns items along the main axis
  • align-items - Aligns items along the cross axis
  • flex-wrap - Specifies whether the flex items should wrap or not

CSS Grid Layout

CSS Grid Layout is a two-dimensional layout system for the web. It allows you to layout items into rows and columns, making it easier to design web pages without having to use floats and positioning.

.container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    grid-gap: 10px;
}

.item {
    background-color: lightblue;
    padding: 20px;
}

Media Queries

Media queries are a key part of responsive web design, allowing you to apply styles based on the characteristics of the device being used to view the content.

@media only screen and (max-width: 600px) {
    body {
        background-color: lightblue;
    }
}

@media only screen and (min-width: 601px) {
    body {
        background-color: lightgreen;
    }
}

Flexible Images

Images in a responsive web design should be flexible to fit their containing elements.

img {
    max-width: 100%;
    height: auto;
}

Viewport

The viewport meta tag ensures that your website is properly scaled on all devices.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

CSS Transitions

CSS transitions allow you to change property values smoothly (over a given duration).

div {
    width: 100px;
    height: 100px;
    background-color: red;
    transition: width 2s;
}

div:hover {
    width: 300px;
}

CSS Animations

CSS animations allow you to animate transitions from one CSS style configuration to another.

@keyframes example {
    from {background-color: red;}
    to {background-color: yellow;}
}

div {
    width: 100px;
    height: 100px;
    background-color: red;
    animation-name: example;
    animation-duration: 4s;
}

Introduction to Geolocation

The HTML5 Geolocation API is used to get the geographical position of a user.

<button onclick="getLocation()">Try It</button>
<p id="demo"></p>

<script>
var x = document.getElementById("demo");

function getLocation() {
    if (navigator.geolocation) {
        navigator.geolocation.getCurrentPosition(showPosition);
    } else {
        x.innerHTML = "Geolocation is not supported by this browser.";
    }
}

function showPosition(position) {
    x.innerHTML = "Latitude: " + position.coords.latitude +
    "<br>Longitude: " + position.coords.longitude;
}
</script>

Introduction to Web Workers

Web Workers allow you to run scripts in background threads, which can improve the performance of your web application.

<button onclick="startWorker()">Start Worker</button>
<button onclick="stopWorker()">Stop Worker</button>
<p id="demo"></p>

<script>
var worker;

function startWorker() {
    if (typeof(Worker) !== "undefined") {
        if (typeof(worker) == "undefined") {
            worker = new Worker("worker.js");
        }
        worker.onmessage = function(event) {
            document.getElementById("demo").innerHTML = event.data;
        };
    } else {
        document.getElementById("demo").innerHTML = "Sorry, your browser does not support Web Workers...";
    }
}

function stopWorker() {
    worker.terminate();
    worker = undefined;
}
</script>

Contents of worker.js:

var i = 0;

function timedCount() {
    i = i + 1;
    postMessage(i);
    setTimeout("timedCount()", 500);
}

timedCount();

Introduction to Web Sockets

Web Sockets provide a full-duplex communication channel over a single TCP connection, enabling real-time data transfer between the server and the client.

<script>

ws.onopen = function() {
    ws.send("Hello, Server!");
};

ws.onmessage = function(event) {
    var msg = event.data;
    alert("Message received: " + msg);
};

ws.onclose = function() {
    alert("Connection closed");
};
</script>

<div id="div1" ondrop="drop(event)" ondragover="allowDrop(event)"></div>
<br>
<div id="drag1" draggable="true" ondragstart="drag(event)">Drag me</div>

<script>
function allowDrop(ev) {
    ev.preventDefault();
}

function drag(ev) {
    ev.dataTransfer.setData("text", ev.target.id);
}

function drop(ev) {
    ev.preventDefault();
    var data = ev.dataTransfer.getData("text");
    ev.target.appendChild(document.getElementById(data));
}
</script>

Introduction to IndexedDB

IndexedDB is a low-level API for client-side storage of significant amounts of structured data, including files/blobs.

<script>
var request = indexedDB.open("MyTestDatabase", 1);

request.onerror = function(event) {
    console.log("Database error: " + event.target.errorCode);
};

request.onsuccess = function(event) {
    console.log("Database opened successfully");
};

request.onupgradeneeded = function(event) {
    var db = event.target.result;
    var objectStore = db.createObjectStore("customers", { keyPath: "id" });

    objectStore.transaction.oncomplete = function(event) {
        var customerObjectStore = db.transaction("customers", "readwrite").objectStore("customers");
        customerObjectStore.add({ id: 1, name: "John Doe", email: "john.doe@example.com" });
    };
};
</script>
